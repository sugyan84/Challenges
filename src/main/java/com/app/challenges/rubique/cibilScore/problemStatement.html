CIBIL Score

Rubique has a unique logic to rate its customers basis the CIBIL score and the RISK score. Following is the algorithm used to rate and sort the customers.


1. The customer with higher CIBIL score will be on the top.
2. If CIBIL score of two customer matches then the customer with lower RISK score will come above.
3. If both CIBIL and RISK score matches, then the customers are sorted in descending order of their names.
4. If all three value matches then it will remain in the same order as input.

Task is to find the Kth customer from the top in the sorted list of customers starting from 1;

NAME contain only alphabets without any space.

Input

T : Total number of test cases. In each test, the first line indicates the number of customers N

The N lines following it contain 3 space separated data  i.e NAME, CIBIL, NEGATIVE INDEX. The n+1th line contains value of K.
  

Output
Print two space separated value i.e NAME and CIBIL of the kth customer
For each test case, output should be in a different line.

Constraints
0<k1000
0T100
0CIBIL800
0RISKScore200

Explanation

In the test case 1 of the sample the sorted order as per the conditions in the problem statement will be - 
Suraj 670 70
Prateek 670 80
Sumit 640 90
Ayush 640 100
Sahil 630 20
Sameer 620 50
So the third entry after sorting is : Sumit 640 90
